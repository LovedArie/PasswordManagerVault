# 08 — Development Case

> **What this is**: A short document describing the **customized** UP for
> *this* project — which artifacts you'll create, when, and which practices
> you'll follow. (Larman, *Applying UML and Patterns*, Ch. 2 §2.12.)
>
> **Why it matters**: Larman is clear (Ch. 2 §2.12) that almost every UP
> artifact is **optional**. The Development Case is where you decide, for
> this project, which ones add real value. Like a pharmacy: pick the
> medicines that match the ailment.

> **This is the file that decides what the rest of the `docs/` folder looks
> like.** Once you fill it in, you can confidently delete artifacts you've
> opted out of.
